# 上傳問題解決

## ❌ 問題總結

### GitHub API 422 錯誤

**錯誤訊息：** `"sha" wasn't supplied.`
**HTTP 狀態碼：** 422

**根本原因：** GitHub API 在建立新檔案時，如果該路徑已存在，必須提供現有檔案的 SHA 值。

---

## 🔧 解決方案

### 方式 1：手動上傳到 GitHub 頁面（推薦，最可靠）

**為什麼推薦：**
1. ✅ **不依賴任何 API** - 直接在瀏覽器中操作
2. ✅ **最簡單** - 拖放檔案即可
3. ✅ **最直觀** - 可以看到上傳進度和結果
4. ✅ **不會遇到 API 問題** - 不需要處理 SHA、認證等技術問題

### 方式 2：手動刪除舊檔案後再上傳

**操作步驟：**
1. **訪問 OpenClawNote 倉庫**
   - URL: https://github.com/jetannku-hash/OpenClawNote
   - 查看是否已有檔案
2. **手動刪除舊檔案**
   - 點擊檔案旁的「...」
   - 選擇「Delete this file」
3. **上傳新檔案**
   - 點擊「Upload files」
   - 拖放 `/home/terryku/.openclaw/workspace/my_obsidian_vault/` 目錄

### 方式 3：使用 curl + base64（進階解決方案）

**問題：** GitHub API 在更新檔案時需要 SHA 值

**解決方式：**
1. 先 GET 檔案資訊獲取 SHA
2. 再使用 SHA 進行更新

```bash
# 獲取檔案 SHA
SHA=$(curl -s \
  -H "Authorization: token YOUR_GITHUB_TOKEN_HERE" \
  -H "Accept: application/vnd.github.v3+json" \
  "https://api.github.com/repos/jetannku-hash/OpenClawNote/contents/FILENAME" | jq -r '.sha')

# 使用 SHA 更新檔案
curl -X PUT \
  -H "Authorization: token YOUR_GITHUB_TOKEN_HERE" \
  -H "Accept: application/vnd.github.v3+json" \
  -H "Content-Type: application/json" \
  -d "{\"message\":\"更新 FILENAME\", \"sha\":\"$SHA\", \"content\":\"$(base64 -w 0 FILE)\"}" \
  "https://api.github.com/repos/jetannku-hash/OpenClawNote/contents/FILENAME"
```
