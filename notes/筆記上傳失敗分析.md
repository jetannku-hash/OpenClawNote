# 筆記上傳失敗分析

## ❌ 問題總結

### 所有上傳嘗試都失敗了

| 嘗試方式 | 錯誤訊息 | 原因 |
|----------|----------|------|
| **upload_notes.sh** | `sha" wasn't supplied` (HTTP 422) | GitHub API 需要提供現有檔案的 SHA |
| **batch_upload_memory.sh** | `sha" wasn't supplied` (HTTP 422) | GitHub API 需要提供現有檔案的 SHA |
| **upload_notes_fixed.sh** | `sha" wasn't supplied` (HTTP 422) | GitHub API 需要提供現有檔案的 SHA |

---

## 🔬 根本原因

### GitHub API 的限制

當檔案路徑已存在於倉庫時，必須提供現有檔案的 SHA 值，否則會返回 HTTP 422 錯誤：

```json
{
  "message": "Invalid request.\n\n\"sha\" wasn't supplied.",
  "documentation_url": "https://docs.github.com/rest/repos/contents#create-or-update-file-contents",
  "status": "422"
}
```

**為什麼需要 SHA？**

1. **防止意外覆蓋** - 系統需要知道你要更新哪個版本
2. **衝突檢測** - 避免 Git 歷史衝突
3. **原子操作** - 確保操作是原子性的

### 方式 2：先 GET SHA 再 PUT（自動化）

**操作流程：**
1. **檢查檔案是否存在**
2. **如果存在：GET 檔案資訊，獲取 SHA**
3. **使用 PUT + SHA 更新檔案**
4. **如果不存在：使用 PUT 新增檔案（不需要 SHA）**

**解決方案腳本：**

```bash
#!/bin/bash
# 支援 SHA 的筆記上傳腳本

REPO="jetannku-hash/OpenClawNote"
SOURCE_DIR="/home/terryku/.openclaw/workspace/my_obsidian_vault"

cd "$SOURCE_DIR"

# 遍歷所有 .md 檔案
find . -name "*.md" -type f | sort | while read -r file; do
    FILENAME=$(basename "$file")
    REL_PATH="${file#./}"
    
    # 檢查檔案是否已存在
    CHECK_EXISTS=$(curl -s \
        -H "Authorization: token YOUR_GITHUB_TOKEN_HERE" \
        -H "Accept: application/vnd.github.v3+json" \
        "https://api.github.com/repos/$REPO/contents/$REL_PATH")
    
    # 解析回應
    if echo "$CHECK_EXISTS" | grep -q '"name"'; then
        # 檔案已存在，獲取 SHA
        SHA=$(echo "$CHECK_EXISTS" | grep -oP '"sha":"[^"]+" | cut -d'"' -f4)
        
        echo "📤 [$total_files] 更新：$FILENAME (SHA: ${SHA:0:7})"
        
        # 使用 SHA 更新檔案
        content=$(cat "$file")
        base64_content=$(echo "$content" | base64 -w 0)
        
        curl -X PUT \
            -H "Authorization: token YOUR_GITHUB_TOKEN_HERE" \
            -H "Accept: application/vnd.github.v3+json" \
            -H "Content-Type: application/json" \
            -d "{
                \"message\": \"更新 $FILENAME\",
                \"sha\": \"$SHA\",
                \"content\": \"$base64_content\"
            }" \
            "https://api.github.com/repos/$REPO/contents/$REL_PATH"
    else
        # 檔案不存在，直接新增
        echo "📤 [$total_files] 新增：$FILENAME"
        
        content=$(cat "$file")
        base64_content=$(echo "$content" | base64 -w 0)
        
        curl -X PUT \
            -H "Authorization: token YOUR_GITHUB_TOKEN_HERE" \
            -H "Accept: application/vnd.github.v3+json" \
            -H "Content-Type: application/json" \
            -d "{
                \"message\": \"新增 $FILENAME\",
                \"content\": \"$base64_content\"
            }" \
            "https://api.github.com/repos/$REPO/contents/$REL_PATH"
    fi
    
    sleep 0.5  # 避免 API 限制
done

echo "🎉 所有筆記上傳完成！"
```

### 方式 3：使用 Git push（最可靠、長期方案）

**優點：**
- ✅ 最可靠、最快速
- ✅ Git 自動處理所有 SHA 和衝突
- ✅ 支援增量推送
- ✅ 之後只需 `git add` 和 `git push`

**操作步驟：**

#### 第一次：初始化 Git 倉庫

```bash
cd /home/terryku/.openclaw/workspace/my_obsidian_vault

# 初始化 Git
git init

# 設定使用者
git config user.name "terryku-hash"
git config user.email "jetannku@gmail.com"

# 添加所有檔案
git add .

# 初始提交
git commit -m "初始化筆記倉庫"

# 添加遠程
git remote add origin https://github.com/jetannku-hash/OpenClawNote.git

# 推送
git push -u origin main
```

#### 之後：每次更新只需

```bash
cd /home/terryku/.openclaw/workspace/my_obsidian_vault

# 添加更新的檔案
git add .

# 提交更新
git commit -m "更新筆記：$(date +'%Y-%m-%d %H:%M')"

# 推送更新
git push
```
