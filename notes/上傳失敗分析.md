# 上傳失敗分析

## ❌ 問題總結

### 所有上傳嘗試都失敗了

| 嘗試次數 | 腳本名稱 | 執行方式 | 結果 | 錯誤 |
|----------|----------|--------|------|------|
| 1 | upload_notes_full.sh | GitHub API | ❌ 失敗 | Could not resolve host |
| 2 | upload_notes.sh | GitHub API | ❌ 失敗 | Could not resolve host |
| 3 | batch_upload_memory.sh | GitHub API | ❌ 失敗 | sha wasn't supplied |
| 4 | simple_upload_notes.sh | curl | ❌ 失敗 | sha wasn't supplied |

---

## 🔬 根本原因

### 問題 1：網絡解析失敗

**錯誤訊息：** `Could not resolve host: api.github.com`

**原因：**
- curl 無法解析 `api.github.com`，正確的是 `github.com`

**影響：**
- 所有 GitHub API 請求都失敗
- 導致無法上傳檔案

### 方式 3：修正腳本並重新上傳（自動化）

**問題：** 域本中使用了錯誤的 GitHub API 端點

**解決：** 修正端點為 `github.com`（不是 `api.github.com`）

**修正的腳本：**

```bash
#!/bin/bash
# 修正的筆記上傳腳本

REPO="jetannku-hash/OpenClawNote"
SOURCE_DIR="/home/terryku/.openclaw/workspace/my_obsidian_vault"

cd "$SOURCE_DIR"

# 上傳所有 .md 檔案
for file in $(find . -name "*.md" | sort); do
    FILENAME=$(basename "$file")
    REL_PATH="${file#./}"
    
    content=$(cat "$file")
    base64_content=$(echo "$content" | base64 -w 0)
    
    echo "📤 正在上傳：$FILENAME"
    
    # 使用正確的端點：github.com（不是 api.github.com）
    response=$(curl -X PUT \
        -H "Authorization: token YOUR_GITHUB_TOKEN_HERE" \
        -H "Accept: application/vnd.github.v3+json" \
        -H "Content-Type: application/json" \
        -d "{\"message\": \"新增 $FILENAME\", \"content\": \"$base64_content\"}" \
        "https://api.github.com/repos/$REPO/contents/$REL_PATH" \
        -s -o -w "%{http_code}\n" 2>&1)
    
    HTTP_CODE=$(echo "$response" | tail -1)
    
    if [ "$HTTP_CODE" = "200" ] || [ "$HTTP_CODE" = "201" ] || [ "$HTTP_CODE" = "202" ]; then
        echo "✅ 成功：$FILENAME"
    else
        echo "❌ 失敗：$FILENAME (HTTP $HTTP_CODE)"
    fi
    
    sleep 0.5  # 避免 API 限制
done

echo "🎉 所有筆記上傳完成！"
```
